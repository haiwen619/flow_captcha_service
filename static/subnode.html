Subnode Status

这是子节点状态页，不提供用户注册和登录。

当前实例用于执行本地有头浏览器打码，并向主节点上报心跳与容量。 如果你要给用户提供门户、充值、API Key 管理，请访问主节点或单机实例的主页。

节点 -- 角色 subnode 状态 检查中
进入管理后台 打开健康检查 查看公开概览

节点职责

  • 执行本地 Playwright + Chromium 打码。
  • 向 master 注册并持续上报心跳。
  • 接收 routed session 的 `finish` / `error` 回调。

不包含的功能

  • 不对外开放用户自注册和登录首页。
  • 不负责用户额度、CDK、Portal API Key 自助管理。
  • 不负责集群调度决策，调度由 master 执行。

排查入口

  • /api/v1/health 查看存活状态。
  • /admin 查看配置、日志和节点执行信息。
  • /api/portal/summary 查看公共运行摘要。

接入说明

对外用户入口应该放在 `master` 或 `standalone` 实例。子节点主页只保留运行状态展示， 目的是让部署人员确认这个节点是否已启动、是否处于 `subnode` 角色，以及浏览器槽位是否正常工作。

如果这里显示的角色不是 subnode，说明当前实例配置可能与预期不一致， 请检查 FCS_CLUSTER_ROLE 或 data/setting.toml。
